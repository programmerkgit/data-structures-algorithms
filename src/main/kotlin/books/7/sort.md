## ソートの判断基準
- Order
- メモリ
- 安定性
[参考](https://qiita.com/drken/items/44c60118ab3703f7727f)

## merge-sort
n個を並び替えるアルゴリズム。nLogN
- log n 個の階層
- それぞれ n の計算量
- メモリ O(n)

## クイックソート
Partition(O(n))を繰り返したもの。log2 nの階層回実行される。

- 平均nlogN. Max O(n^2)やスタックオーバーフローのリスク
- 内部ソート

## 計数ソート
- O(n + K)
- 最大値に比例するメモリ
- 安定

## ヒープソート
安定性ではマージソート、メモリとスピードではクイックソート
- 非安定
- n log n 


## partition
配列を x以上とそれ以下に分割
- O(n)
